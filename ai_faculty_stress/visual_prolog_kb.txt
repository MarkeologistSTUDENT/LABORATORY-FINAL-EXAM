Visual Prolog KB template and integration notes

This file contains a simple, copy-paste friendly Visual Prolog skeleton and notes to integrate the Python exporter output (`predicted_stress_facts.pl`).

Important: Visual Prolog (the commercial IDE) uses its own project and source file formats. Below is a minimal, easy-to-follow approach you can paste into a new Visual Prolog "Console Application" project and adapt.

1) Convert facts file into Visual Prolog database

The Python exporter already writes `predicted_stress_facts.pl` using classic Prolog facts of the form:

predicted_stress('F001','Medium').

Visual Prolog does not use `consult/1` the same way SWI-Prolog does. The easiest approach is:
- Open `predicted_stress_facts.pl` in a text editor and copy the fact lines into a new Visual Prolog source file (see step 2) or
- Keep the file as-is and write a tiny Visual Prolog parser to read each line and assert facts at runtime.

2) Minimal Visual Prolog KB (recommended: paste into a project source file)
